docker build -t zcu102-emulator .

**2. Prepare your Workspace:**
Create a folder on your host machine (e.g., `./my_project`) and place your compiled Linux files (usually generated by PetaLinux or Yocto) and your R5 firmware (`npc_firmware.elf`) inside it.

**3. Run the Emulator:**
Start the container, mounting your project files into the `/workspace` directory:
```bash
docker run -it --rm \
    -v $(pwd)/my_project:/workspace \
    -p 4444:4444 -p 5555:5555 \
    zcu102-emulator
*(If your filenames differ from the defaults in the script, you can override them by adding environment variables to the docker run command, e.g., `-e LINUX_KERNEL="zImage"`).*

**4. Connect to the Serial Terminals:**
Open two new terminal windows on your host machine:
* **For A53 Linux CLI:** `telnet localhost 4444`
* **For R5 NPC Firmware CLI:** `telnet localhost 5555`

### A Note on ZynqMP Emulation Architecture:
* **CPU Mapping:** In Xilinx QEMU, the `xlnx-zcu102` machine defines the Quad-core A53 as CPUs `0, 1, 2, 3` and the Dual-core R5 as CPUs `4, 5`. The `-device loader,file=...,cpu-num=4` flag specifically injects your NPC application directly into the first R5 core, bypassing the need for complex First Stage Boot Loader (FSBL) setups.
* **Shared Memory:** Because both architectures are emulated in the same monolithic instance, any physical memory addresses (e.g., OCM at `0xFFFC0000` or DDR) written to by the R5 will be immediately visible to the A53, allowing you to seamlessly test your physical hardware interfaces and IPC mechanisms.
